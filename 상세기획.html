지도 기반 파티 서비스 상세기획

상위기획의 방향을 바탕으로 MVP에서 실제로 구현해야 할 화면, 상태, 정책, 사용자 흐름, 데이터 요구사항을 정의한다.

상세기획 기준

사용자가 처음 들어와 파티를 만들고, 멤버를 모으고, 장소를 제안하고, 대화 안에서 결정을 끝내는 전체 루프가 끊기지 않아야 한다.

MVP 범위 IA와 화면 핵심 플로우 정책과 상태 데이터/API

1. MVP 상세 범위

MVP는 특정 타깃군으로 과하게 좁히지 않고, 좌표/POI 기반 라이브 파티 경험을 먼저 검증한다. 사용자는 미래 약속을 관리하는 것이 아니라, 지도 위 특정 좌표에서 지금 열린 파티를 발견하고 입장해 채팅으로 합류하는 흐름을 경험한다.

첫 버전의 성공 기준은 "사람들이 카톡방 대신 이 파티방에서 장소를 정했는가"다.

라이브 파티 원칙

기본 인식: 사용자는 이 서비스를 일정 관리 도구가 아니라 지금 살아 있는 파티방으로 받아들여야 한다.
시간 범위: MVP 파티는 기본적으로 오늘 또는 곧 움직일 모임을 우선한다. 사용자가 파티명에 미래 날짜를 섞어 쓰는 것은 막지 않지만, 날짜 투표나 캘린더 관리는 제공하지 않는다.
노출 방식: 홈과 파티 상세에서는 "방금 열림", "지금 논의 중", "모집 중", "집결 중"처럼 현재성이 느껴지는 상태를 우선 노출한다.
시스템 UI 킥: iOS Live Activities의 Lock Screen, Dynamic Island, compact, minimal, expanded 표현을 서비스의 핵심 UI 레퍼런스로 삼는다. 파티는 앱 안에만 있는 카드가 아니라 시스템 상단에 살아 있는 상태로 느껴져야 한다.
자동 정리: 오늘 기준 노출 기간이 지나면 파티는 지도 발견 목록에서 내려간다. 단, 그룹채팅과 파티 기록은 소멸시키지 않는다.

오늘 우선 노출 정책

지도 우선순위: 지도에서 특정 좌표나 POI를 눌렀을 때 오늘 생성됐거나 오늘 활동 중인 live 파티를 최상단에 노출한다.
미래 날짜 혼입: "7/6 저녁 ㄱㄱ"처럼 파티명에 미래 날짜가 들어가도 MVP에서는 일정 객체로 강제 변환하지 않는다. 제목 텍스트로 보존하고 내부 분석 후보로만 쌓는다.
지도 노출 만료: 파티의 map_visible_until이 지나면 지도와 위치별 파티 목록에서 더 이상 발견되지 않는다.
채팅 보존: 지도에서 내려간 파티도 참여자들의 그룹채팅 목록에는 남는다. 사용자는 채팅을 계속 열람하거나 직접 나갈 수 있다.
상태 분리: 지도 발견 상태와 그룹채팅 생존 상태를 분리한다. 지도에서 사라졌다는 것이 파티 삭제나 채팅방 삭제를 의미하지 않는다.

포함 기능

영역 기능 완료 기준
인증/계정 카카오 로그인, 기본 프로필 설정, 선택 인증 사용자가 카카오로 가입하고 필요 시 학교/회사 등 선택 인증을 파티 접근 조건 검증에 사용한다.
파티 파티 생성, 즉시 입장 초대 URL, 참여 조건, 파티용 프로필 호스트가 파티를 만들고 링크를 공유하면 초대받은 사용자가 조건 확인 후 입장한다.
지도/파티 탐색 네이버 지도 표시, POI 클릭, 지도 롱프레스, 해당 위치 파티 목록 사용자가 POI를 누르거나 지도 특정 지점을 꾹 누르면 그 좌표 또는 POI에 생성된 라이브 파티 목록이 열린다.
채팅 파티별 그룹채팅, 참여 시스템 메시지, 위치 카드 사용자가 위치 기반 파티에 입장하면 채팅방에서 바로 대화와 합류가 가능하다.
의사결정 파티 입장, 참여 의사 표현, 채팅 합류 멤버가 해당 위치에서 열린 파티를 보고 들어갈지 결정할 수 있다.

제외 기능

공개 탐색: 주변 공개 파티 피드와 추천은 2차로 미룬다.
실시간 위치: 멤버 현재 위치 공유는 개인정보 리스크가 커서 MVP에서 제외한다.
장소 후보 의사결정: 여러 장소를 후보로 올리고 투표하는 기능은 MVP에서 제외한다.
미래 일정 관리: 날짜 투표, 캘린더 연동, 반복 일정, 리마인드는 MVP에서 제외한다.
복잡한 모더레이션: 신고, 차단, 제재 자동화는 운영 정책 문서는 정의하되 MVP 구현은 최소화한다.

초기 파티 시나리오

파티 제목은 사용자의 실제 목적이 바로 드러나는 문장형 제목을 기본으로 한다. MVP의 파티는 "어디에서 무엇을 같이 할 사람을 구하는가"를 먼저 보여준다. 지도는 후보 장소를 정하는 도구가 아니라, 위치에 붙은 라이브 파티를 발견하는 진입점이다.

2. IA와 화면 구성

서비스의 첫 화면은 빈 지도나 기능 소개가 아니라, 사용자가 바로 움직일 수 있는 파티 맥락을 보여줘야 한다.

파티 만들기 창

파티 만들기는 지도에서 좌표나 POI를 누른 뒤 열리는 바텀시트 또는 모달이다. 프로토타입에서도 이 창을 반드시 거쳐야 하며, 생성 완료 후 곧바로 그룹채팅으로 넘기지 않고 파티장이 다음 행동을 선택하게 한다.

단계 입력/표시 항목 정책
1. 기준 위치 확인 장소명 또는 좌표명, 주소, 지도 미니 프리뷰 지도에서 선택한 위치를 기본값으로 고정한다. MVP에서는 생성 중 위치 변경을 제공하지 않는다.
2. 파티명 입력 파티 제목, 추천 제목, 글자 수 카운터, 파티 프로필 사진 제목은 필수다. 예: 샤이니 콘서트 2명, 같이 가평 놀러가실분, 여의도 불꽃축제 2명. 파티 프로필 사진은 선택 입력이며 미등록 시 기본 파티 이미지 또는 장소 기반 기본 이미지를 사용한다.
3. 공개/비공개 설정 공개파티, 비공개파티 공개파티는 지도 목록에서 즉시 입장 가능하다. 비공개파티는 승인, 비밀번호, 즉시 입장 초대 URL 중 허용 방식을 정한다.
4. 접근 조건 설정 제한 없음, 내가 인증한 대학교만 허용, 직장 인증 사용자만 허용 공개/비공개와 별개로 적용된다. 사용자가 조건을 만족하지 않으면 입장 전 인증 안내를 보여준다.
5. 파티용 프로필 확인 닉네임, 프로필 이미지 계정 신원과 파티 프로필은 분리한다. 기본 프로필을 쓰되 파티별 수정이 가능하다.
6. 생성 완료 파티 프로필 사진, 파티명, 기준 위치, 현재 입장 인원 1명, 초대 URL, 동네 초대 가능 사용자 리스트, 채팅 열기 버튼 생성 직후에는 초대 후보와 다음 행동을 보여준다. 파티장은 채팅으로 이동하거나 초대 URL을 공유하거나 동네 사용자에게 초대를 보내거나 지도 목록으로 돌아갈 수 있다.
파티 프로필 사진: 파티 자체의 대표 이미지이며 파티장의 개인 프로필 이미지와 분리한다.
노출 위치: 위치 파티 카드, 톡 채팅방 행, 파티 상세 상단, 초대 화면, Live Activity형 요약에 사용할 수 있다.
수정 권한: 생성 이후에는 현재 파티장만 파티 설정에서 프로필 사진을 변경할 수 있다.
생성 완료 제외: 파티 생성 완료 화면에는 Live Activity형 캡슐을 별도 요소로 노출하지 않는다. 생성 완료 화면은 초대와 다음 행동 선택에 집중한다.

동네 초대 가능 사용자 리스트

파티 생성 완료 화면에서는 해당 위치의 동네 기준으로 초대할 수 있는 사용자를 보여준다. 이 리스트는 생성 완료 화면 전용이 아니라 각 그룹채팅 안의 초대 화면에서도 같은 컴포넌트와 같은 조회 정책으로 재사용한다.

항목 정책
노출 기준 파티 기준 위치의 동네 또는 근접 생활권에 속하고, 초대 수신을 허용한 사용자만 보여준다.
입장 조건 필터 학교/직장 인증 조건이 있는 파티는 해당 조건을 충족했거나 충족 가능 안내를 받을 수 있는 사용자만 후보로 노출한다.
제외 대상 이미 파티에 참여 중인 사용자, 이미 초대를 받은 사용자, 동일 파티 초대를 3회 거절해 차단된 사용자는 제외한다.
카드 정보 프로필 이미지, 닉네임, 동네명 또는 거리감, 온라인 여부, 초대 버튼을 보여준다. 실명, 이메일, 카카오 계정 식별자는 노출하지 않는다.
초대 버튼 누르면 즉시 입장 초대 URL 또는 파티 초대 알림을 발송한다. 발송 후 버튼은 초대 완료 상태로 바뀐다.
빈 상태 "지금 초대할 수 있는 동네 사용자가 없어요"를 보여주고, 초대 URL 공유 액션을 유지한다.
재사용 위치: 파티 생성 완료 화면, 그룹채팅 상단 초대 버튼, 그룹채팅 멤버 목록의 초대 탭에서 같은 리스트를 사용한다.
초대 허용: 사용자가 프로필/개인정보 설정에서 파티 초대 허용을 꺼두면 이 리스트에 노출하지 않는다.
동네 기준: POI는 POI 주소의 동네를 기준으로 하고, 좌표 파티는 reverse geocoding으로 얻은 동네 또는 서비스가 계산한 근접 생활권을 기준으로 한다.
정렬: 같은 동네, 최근 온라인, 파티 접근 조건 충족, 최근 초대 피로도가 낮은 사용자 순으로 정렬한다.

  • LocationPartyCard: 파티명, 파티명 우측의 연한 참가 인원 숫자와 상태 아이콘, 생성 시각, 우측 CTA

지도 POI 심볼 표시

지도 위 POI는 상호명 텍스트를 상시 노출하기보다, 네이버 지도 북마크 기호처럼 작은 심볼 중심으로 제공한다. 지도는 한눈에 위치와 파티 존재 여부를 파악하는 영역이고, 장소명과 상세 정보는 사용자가 선택한 뒤 패널에서 보여준다.

대상 지도 위 표시 선택 후 표시
POI 북마크/핀형 심볼, 필요 시 live 파티 수 작은 숫자 장소명, 주소, 열린 파티 수, 파티 목록
좌표 기반 위치 좌표 핀 또는 임시 선택 심볼 좌표 기반 지역명, 주소 후보, 열린 파티 수, 파티 목록
파티가 있는 위치 기본 POI 심볼보다 강조된 live 심볼 또는 숫자 해당 위치의 live 파티 목록
상호명 비노출: 지도 위에는 상호명/장소명을 기본 라벨로 계속 노출하지 않는다. 텍스트가 많아져 지도 탐색성이 떨어지는 것을 막는다.
선택 후 정보: 사용자가 심볼을 누르면 위치별 파티 목록 패널에서 장소명 또는 좌표명을 보여준다.
접근성: 화면에는 심볼만 보여도 접근성 라벨에는 장소명, 열린 파티 수, 공개/비공개 여부 요약을 포함할 수 있다.
시각 계층: 파티가 없는 일반 POI보다 live 파티가 있는 POI/좌표 심볼을 더 강하게 표시한다.

지도 현재 위치 이동 버튼

지도 화면에는 사용자의 GPS 현재 위치로 지도를 이동시키는 내 위치 버튼을 제공한다. 사용자가 지도를 탐색하다가 언제든 현재 위치 주변의 파티 맥락으로 돌아올 수 있게 한다.

상태 동작 정책
위치 권한 허용 버튼을 누르면 GPS 좌표 기준으로 지도 중심을 이동한다. 이동 후 현재 위치 주변의 지도 노출 가능 live 파티를 다시 조회한다.
위치 권한 미허용 버튼을 누르면 위치 권한 요청 또는 설정 이동 안내를 보여준다. 권한이 없어도 지도 탐색, 검색, POI/좌표 선택은 계속 가능하다.
GPS 실패 현재 위치를 확인할 수 없다는 안내를 보여준다. 마지막으로 확인된 위치가 있으면 보조 액션으로 이동할 수 있게 한다.
버튼 위치: 지도 우측 하단의 파티 만들기 버튼과 겹치지 않는 위치에 고정한다.
시각 형태: 텍스트 라벨보다 익숙한 위치/타깃 아이콘을 기본으로 하고, 접근성 라벨은 "내 위치로 이동"으로 둔다.
개인정보: 내 위치 버튼은 지도 중심 이동과 주변 파티 조회에만 사용한다. 다른 사용자에게 현재 위치를 공유하지 않는다.

내 위치 표시

지도 위 현재 위치는 사용자 본인 화면에서만 보이는 보조 표시다. 다른 사용자에게 공유하거나 파티 멤버 위치로 노출하지 않는다.

구현 환경 표시 방식 정책
웹/프로토타입 HTML5 Geolocation으로 얻은 좌표에 현재 위치 마커와 반투명 원형 Circle 오버레이를 직접 그린다. 정확도 반경이 있으면 원의 radius에 반영하고, 없으면 기본 작은 반경을 사용한다.
네이티브 앱 Naver Maps SDK의 LocationOverlay 사용을 우선 검토한다. SDK가 제공하는 현재 위치 아이콘, 방향, 그림자, 원형 표시를 활용할 수 있으면 네이티브 기본 동작을 따른다.
권한 없음/실패 현재 위치 마커와 원을 표시하지 않는다. 지도 탐색, 현 위치로 검색, POI/좌표 선택은 계속 가능하게 유지한다.
시각 표현: 현재 위치는 작은 점 또는 타깃형 마커와 반투명 원으로 표현한다. 파티 마커보다 시각적 우선순위가 낮아야 한다.
정확도 원: 원은 GPS 정확도 또는 대략적 위치 범위를 나타내는 용도이며, 검색 반경 1km 원과 혼동되지 않게 색상과 투명도를 다르게 둔다.
갱신: 위치가 갱신되면 마커와 원 위치만 갱신하고, 자동으로 지도를 계속 따라가지는 않는다. 사용자가 내 위치 버튼을 누를 때 지도 중심을 이동한다.

지도 이동 후 현 위치로 검색

사용자가 지도를 드래그하거나 줌을 변경해 탐색 영역이 바뀌면, 현재 지도 중심 또는 화면 bounds 기준으로 서버의 POI와 지도 노출 가능 파티를 다시 조회할 수 있어야 한다. 이 기능은 GPS 현재 위치로 이동하는 버튼과 다르며, 사용자가 보고 있는 지도 영역을 기준으로 검색한다.

상황 UI 정책
사용자가 지도를 이동/확대/축소 지도 상단 또는 중앙 하단에 "현 위치로 검색" 버튼 노출 버튼은 지도 카메라가 마지막 조회 영역에서 의미 있게 벗어났을 때만 보여준다.
버튼 선택 로딩 상태 후 POI/파티 마커 갱신 현재 지도 중심 좌표, 줌 레벨, bounds, 검색 반경을 서버로 보내 POI와 live 파티 요약을 받는다.
검색 결과 없음 빈 상태 토스트 또는 작은 안내 지도 조작은 유지하고, "이 위치에 파티 만들기" 흐름은 계속 제공한다.
네트워크 실패 재시도 가능한 오류 안내 기존 마커를 즉시 지우지 않고, 마지막 성공 조회 결과를 유지한다.
서버 기준: 클라이언트는 네이버 지도에서 얻은 카메라 중심/bounds만 전달하고, 노출할 POI와 파티 요약은 우리 서버 응답을 기준으로 그린다.
검색 반경: MVP 기본 반경은 지도 중심 기준 1km로 둔다. 지도 줌/bounds에 따라 서버가 500m~3km 사이에서 보정하며, 3km를 넘는 광역 검색은 제공하지 않는다.
반경 보정: 사용자가 아주 가까이 확대한 경우에도 최소 500m는 조회해 빈 결과를 줄이고, 멀리 축소한 경우에는 최대 3km로 제한해 라이브/동네 감각을 유지한다.
결과 제한: 한 번의 현 위치 검색 결과는 POI/좌표 그룹 기준 최대 50개까지 내려주고, 지도 노출 가능 live 파티가 있는 위치를 우선 정렬한다.
중복 호출 방지: 지도 이동 중에는 자동 호출하지 않고, 사용자가 "현 위치로 검색"을 누를 때 조회한다. 필요 시 마지막 조회 bounds와 비교해 버튼 노출 여부를 결정한다.
POI와 파티 분리: 응답에는 서버가 관리하는 POI 요약과 각 POI/좌표에 붙은 지도 노출 가능 live 파티 수를 함께 내려준다.

위치별 파티 목록 패널

POI를 클릭하거나 지도 특정 지점을 꾹 누르면 위치별 파티 목록 패널이 열린다. 이 명칭은 내부 화면명이며 사용자에게 타이틀로 노출하지 않는다. 사용자에게는 선택한 장소명 또는 좌표명이 상단 타이틀로 보인다. 기본은 하단에 걸친 작은 패널이지만, 별도 확장 버튼 없이 사용자가 위로 드래그하면 네이버 지도 앱처럼 자연스럽고 부드럽게 화면을 차지하는 전체 페이지 상태로 전환된다.

상태 진입/조작 화면 구성
collapsed panel POI 클릭 또는 지도 롱프레스 시 기본 진입 장소명/좌표명, 주소, 열린 파티 수, 상위 2~3개 파티 카드, 파티 만들기 버튼
expanded panel 패널을 위로 드래그 더 많은 LocationPartyCard, 정렬/필터, 공개/비공개 자물쇠 아이콘, 학생/직장 인증 아이콘
full page 패널을 끝까지 위로 드래그 상단 앱바, 장소명/좌표명 타이틀, 전체 파티 목록, 위치 요약, 파티 만들기 고정 버튼
확장 방식: 목록 확장을 위한 별도 버튼은 두지 않는다. 위로 드래그하는 제스처만으로 collapsed, expanded, full page 상태가 자연스럽게 이어져야 한다.
상단 컨트롤: full page 상태에서는 왼쪽 위에 < 뒤로가기 버튼, 오른쪽 위에 X 닫기 버튼을 제공한다.
사용자 노출 타이틀: "위치별 파티 목록"이라는 문구는 사용자에게 보여주지 않는다. full page 상단 타이틀은 장소명, POI명, 또는 좌표 기반 지역명으로 표시한다.
POI 없는 좌표: 지도 롱프레스로 선택한 지점에 POI가 없으면 reverse geocoding으로 도로명, 지번, 동네명 중 가능한 값을 가져와 좌표 기반 지역명으로 표시한다.
기준 위치 출처: 기준 위치는 POI 클릭으로 만들어진 위치와 지도 롱프레스로 만들어진 좌표 위치를 구분해 저장한다.
뒤로가기: < 버튼은 full page에서 이전 패널 상태로 돌아간다. 이전 상태가 없으면 collapsed panel로 축소한다.
닫기: X 버튼은 패널을 완전히 닫고 지도 화면으로 돌아간다.
아래 드래그: full page 또는 expanded 상태에서 아래로 드래그하면 다시 collapsed panel 상태로 축소된다.
컨텍스트 유지: 확장/축소 중에도 선택한 POI, 정렬, 스크롤 위치, 파티 목록 데이터는 유지한다.

위치 파티 카드 아이콘 정책

특정 좌표 또는 POI의 파티 리스트에서는 공개 여부와 인증 조건을 텍스트 배지로 노출하지 않는다. 사용자는 카드의 아이콘만 보고 상태를 빠르게 구분하며, 접근성 라벨과 툴팁에는 의미를 유지한다.

요소 표시 방식 정책
파티명 LocationPartyCard 첫 줄의 주 텍스트 긴 파티명은 한 줄 말줄임 처리한다.
현재 참가 인원 파티명 오른쪽에 연한 회색 숫자 톡 채팅방 행과 동일하게 별도 칩, 배지, 박스 배경을 사용하지 않는다.
상태 정보 참가 인원 숫자 오른쪽에 자물쇠, 학교/직장 인증 등 아이콘 파티명, 참가 인원 숫자, 상태 아이콘이 같은 행에 이어져야 한다. 텍스트 칩으로 만들지 않고 아이콘 메타 정보로만 제공한다.
CTA 카드 우측 끝 버튼 공개파티는 "참가", 비공개 승인 방식은 "참가요청", 비밀번호 방식은 "비밀번호", 초대 URL로 진입한 경우는 "바로 참가"를 표시한다.
보조 정보 생성 시각, 위치 보조 정보, 최근 활동 요약 참가 인원과 접근 조건을 보조 정보 줄에 중복 표기하지 않고, 우측 CTA와 시각적으로 충돌하지 않게 한다.
항목 조건 화면 표기 접근성 라벨
공개 여부 visibility = public 열린 자물쇠 아이콘 공개파티
공개 여부 visibility = private 닫힌 자물쇠 아이콘 비공개파티
인증 조건 access_condition_type = same_school 학교/학사모 아이콘 내가 인증한 대학교만 허용
인증 조건 access_condition_type = company_verified 직장/브리프케이스 아이콘 직장 인증 사용자만 허용
인증 조건 access_condition_type = none 아이콘 없음 제한 없음
레이블 제거: 위치 파티 리스트 카드 안에는 "공개", "비공개", "학생 인증", "직장 인증" 같은 텍스트 레이블을 표시하지 않는다.
칩 제거: LocationPartyCard에서는 참가 인원, 공개 여부, 인증 조건을 칩으로 표시하지 않는다. 참가 인원은 파티명 우측 숫자, 조건은 아이콘으로 통일한다.
아이콘 위치: 상태 아이콘은 파티명 우측의 참가 인원 숫자 옆에 붙인다. 자물쇠 아이콘을 먼저 두고, 인증 조건 아이콘은 그 오른쪽에 나란히 둔다.
CTA 위치: CTA는 LocationPartyCard 우측 끝에 고정한다. 왼쪽 정보 영역이 길어지면 파티명만 말줄임 처리하고 CTA는 줄어들지 않는다.
의미 확인: 사용자가 아이콘을 길게 누르거나 hover할 수 있는 환경에서는 접근성 라벨과 같은 짧은 설명을 툴팁으로 제공한다.

3. 핵심 사용자 플로우

신규 사용자 가입

  1. 사용자가 온보딩 화면에서 카카오로 시작하기를 선택한다.
  2. 카카오 인증이 완료되면 서비스가 기존 가입 여부를 확인한다.
  3. 신규 사용자라면 필수 약관, 개인정보 처리방침, 위치 기반 서비스 약관 동의를 받는다.
  4. 카카오 프로필을 기본값으로 가져오되, 사용자가 서비스 표시 닉네임과 프로필 이미지를 수정할 수 있게 한다.
  5. 동네 기반 초대 리스트 노출 여부를 설정한다. 기본값은 OFF 또는 명시적 동의 전 미노출이다.
  6. 위치 권한을 요청한다. 거절해도 가입은 완료되지만 지도는 검색/수동 이동 중심으로 사용한다.
  7. 학교/직장 인증은 선택 단계로 제안하고, 건너뛰면 홈으로 진입한다.

기존 사용자 로그인

  1. 사용자가 카카오 로그인을 선택한다.
  2. 서비스가 카카오 계정과 연결된 기존 User를 찾고 세션을 발급한다.
  3. 필수 약관이 개정되어 재동의가 필요하면 홈 진입 전에 동의 화면을 보여준다.
  4. 초대 URL에서 진입한 경우 로그인 완료 후 해당 초대장 검증 화면으로 복귀한다.
  5. 마지막으로 참여 중이던 파티 또는 홈 지도 화면으로 이동한다.

학교/직장 이메일 인증

  1. 사용자가 인증 관리 또는 인증이 필요한 파티 입장 화면에서 이메일 인증을 시작한다.
  2. 학교 인증은 학교 이메일 도메인을, 직장 인증은 회사 이메일 도메인을 입력한다.
  3. 서비스가 이메일 형식과 지원 가능한 조직 도메인인지 확인한 뒤 인증 코드를 발송한다.
  4. 사용자가 인증 코드를 입력하면 VerifiedIdentity를 verified 상태로 저장한다.
  5. 파티 입장 중 인증을 시작한 경우 인증 완료 후 원래 파티 입장 화면으로 복귀한다.
  6. 인증 정보는 입장 조건 검증에만 사용하고, 파티 안에는 이메일/실명/회사명이 직접 노출되지 않는다.

인증 상태와 예외

상황 사용자 안내 정책
인증 코드 만료 인증 시간이 만료됐어요. 코드를 다시 받아주세요. 기존 pending 인증은 만료 처리하고 재발송을 허용한다.
지원하지 않는 이메일 도메인 아직 지원하지 않는 학교/회사 이메일이에요. 인증 요청은 막고, 도메인 추가 요청을 받을 수 있다.
파티 조건 미충족 이 파티는 인증된 멤버만 참여할 수 있습니다. 파티 요약은 보여주되 채팅/멤버 상세는 입장 전까지 노출하지 않는다.
인증 만료 인증을 다시 확인해주세요. 만료된 인증은 접근 조건 검증에 사용할 수 없고 재인증을 요구한다.

파티 생성

  1. 사용자가 지도에서 POI를 누르거나 파티를 열고 싶은 특정 지점을 꾹 누른다.
  2. 해당 위치의 파티 목록 드래그 확장 패널이 열린다.
  3. 사용자가 이 위치에 파티 만들기를 누른다.
  4. 서비스가 사용자의 지도 노출 중인 파티 수를 확인한다.
  5. 지도 노출 중인 파티가 이미 3개이면 추가 생성을 막고 기존 파티 정리 안내를 보여준다.
  6. 파티명을 입력하거나 추천명을 사용한다.
  7. 원하면 파티 프로필 사진을 추가한다. 추가하지 않으면 기본 파티 이미지를 사용한다.
  8. 파티 유형을 선택한다: 공개파티 또는 비공개파티.
  9. 비공개파티인 경우 입장 승인, 비밀번호, 초대장 즉시 입장 중 허용할 방식을 정한다.
  10. 필요한 접근 조건을 지정한다: 제한 없음, 내가 인증한 대학교만 허용, 직장 인증 사용자만 허용.
  11. 파티용 닉네임과 프로필 이미지를 확인한다.
  12. 선택한 위치에 파티와 채팅방이 생성되고 필요 시 초대장이 발급된다.
  13. 생성 완료 창에서 파티 프로필 사진, 현재 입장 인원 1명, 초대 URL, 동네 초대 가능 사용자 리스트, 채팅 열기, 지도 목록으로 돌아가기를 보여준다.
  14. 지도에는 해당 위치의 라이브 파티 수가 갱신된다.

지도에서 파티 발견

  1. 사용자가 지도에서 POI를 누르거나 특정 좌표를 꾹 누른다.
  2. 서비스가 해당 위치 반경 또는 POI 기준으로 생성된 active/live 파티를 조회한다.
  3. 위치별 파티 목록 드래그 확장 패널이 열린다.
  4. 사용자가 패널을 위로 드래그하면 별도 버튼 없이 자연스럽게 전체 페이지로 확장된다.
  5. full page 상태에서는 왼쪽 위 < 뒤로가기와 오른쪽 위 X 닫기를 제공한다.
  6. 사용자가 아래로 드래그하면 다시 하단 패널 상태로 축소된다.
  7. 사용자가 파티 제목, 파티명 우측의 연한 참가 인원 숫자와 상태 아이콘, 생성 시각, 우측 CTA를 확인한다.
  8. 입장 가능한 파티의 우측 CTA를 선택한다.
  9. 공개파티는 파티용 닉네임과 프사를 확인한 뒤 채팅방에 바로 입장한다.
  10. 비공개파티는 설정에 따라 입장 요청을 보내거나 비밀번호를 입력한다.
  11. 입장 요청 방식은 파티장 승인 전까지 대기 상태가 되고, 비밀번호 방식은 일치하면 바로 입장한다.

초대 URL로 파티 참여

  1. 사용자가 파티장이 만든 즉시 입장 초대 URL을 연다.
  2. 로그인하지 않은 경우 카카오 로그인으로 이동한다.
  3. 서비스가 초대장 유효성, 사용 횟수, 만료 여부, 파티 요구 인증을 확인한다.
  4. 인증이 부족하면 필요한 인증 절차를 안내한다.
  5. 사용자가 파티용 닉네임과 프사를 확인한다.
  6. 비공개파티라도 유효한 즉시 입장 초대 URL이면 승인이나 비밀번호 없이 채팅방에 자동 입장한다.
  7. 채팅에 참여 시스템 메시지가 기록된다.

파티 입장 후 합류

  1. 사용자가 위치 기반 파티에 입장한다.
  2. 채팅 상단에 파티 기준 위치 카드가 고정되어 보인다.
  3. 사용자가 채팅으로 참여 의사를 밝힌다.
  4. 멤버들이 현재 상황, 집결 여부, 이동 여부를 대화한다.
  5. 호스트가 필요하면 파티 상태를 모집 중, 이동 중, 종료로 변경한다.
  6. 파티가 종료되거나 만료 시간이 지나면 지도 목록에서 내려간다.

톡 채팅방 행 메타 표시

톡 탭의 채팅방 목록에서는 현재 참석 인원을 별도 칩으로 표시하지 않는다. 카카오톡 단체 채팅방처럼 파티명 오른쪽에 연한 숫자로 붙이고, 해당 채팅방의 알림 on/off 상태도 그 오른쪽에 작은 아이콘으로 붙인다.

요소 표시 방식 정책
파티명 채팅방 행 첫 줄의 주 텍스트 긴 파티명은 한 줄 말줄임 처리한다.
현재 참석 인원 파티명 오른쪽에 연한 회색 숫자 별도 칩, 배지, 박스 배경을 사용하지 않는다. 예: 샤이니 콘서트 2명 4
채팅방 알림 상태 참석 인원 숫자 오른쪽에 작은 종/음소거 아이콘 알림 ON은 기본 상태이므로 필요하면 작은 종 아이콘을 표시하고, OFF는 음소거 종 아이콘으로 표시한다.
보조 정보 둘째 줄에 최근 메시지, 시간, 읽지 않은 수 참석 인원과 알림 상태는 둘째 줄이나 별도 칩으로 내려보내지 않는다.
시각 우선순위: 파티명이 가장 강하고, 참석 인원 숫자는 더 연한 색과 작은 크기로 보여준다.
아이콘 위치: 알림 상태 아이콘은 파티명 행에서 참석 인원 숫자 바로 오른쪽에 배치한다.
레이아웃 안정성: 파티명이 길어도 참석 인원 숫자와 알림 아이콘은 오른쪽에 유지하고, 파티명만 말줄임 처리한다.

채팅방 진입 헤더

사용자가 각 채팅방에 진입했을 때 상단 헤더는 대화 영역을 방해하지 않도록 작고 안정적으로 구성한다. 파티명은 가운데 정렬하고, 참가 중 인원은 우측에 연한 숫자로 표시한다.

요소 표시 방식 정책
뒤로가기 좌측 상단 아이콘 버튼 톡 목록으로 돌아간다.
파티명 상단 중앙 정렬, 기존보다 작은 폰트 채팅방 내부에서는 파티명을 과하게 크게 쓰지 않는다. 긴 파티명은 한 줄 말줄임 처리한다.
참가 중 인원 우측 상단, 파티명보다 연한 색의 숫자 별도 칩이나 배지로 만들지 않는다. 예: 4
메뉴 참가 인원 숫자 오른쪽 또는 우측 끝 아이콘 멤버 목록, 초대, 파티 설정으로 진입한다. 파티장에게만 운영 액션을 노출한다.
정렬: 파티명은 화면 중앙 기준으로 정렬한다. 우측 숫자와 메뉴 때문에 파티명이 밀려 보이지 않도록 헤더 영역을 고정 계산한다.
색상: 참가 중 인원 숫자는 보조 정보이므로 파티명보다 낮은 대비로 표시한다.
칩 금지: 채팅방 내부 헤더에서도 참가 인원을 별도 칩, 배지, 박스 형태로 표시하지 않는다.

그룹채팅 목록 제스처

그룹채팅 목록은 카카오톡처럼 스와이프 액션을 제공한다. 제스처는 채팅방 안이 아니라 채팅방 목록 행에서만 동작한다.

제스처 노출 액션 정책
우측으로 슬라이드 알림 끄기 / 알림 켜기 현재 채팅방의 알림 상태를 토글한다. 알림이 꺼져도 Live Activity형 진행 상태는 별도 정책에 따라 유지될 수 있다.
좌측으로 슬라이드 나가기 나가기 버튼을 누르면 확인 다이얼로그를 띄운다. 멤버가 "예"를 선택해야 파티에서 나간다. 호스트는 MVP에서 바로 나갈 수 없고, 파티 종료 또는 방장 넘기기 안내를 보여준다.
알림 상태 표시: 채팅방별 알림 on/off 상태는 톡 채팅방 행의 참석 인원 숫자 오른쪽에 작은 종/음소거 아이콘으로 표시한다.
아이콘 위치: 음소거 아이콘을 파티명 왼쪽에 따로 두지 않는다. 파티명, 참석 인원 숫자, 알림 아이콘이 같은 행에 이어져야 한다.
나가기 확인 문구: "정말 나가시겠습니까?"
확인 버튼: 예. 파티에서 나가고 그룹채팅 목록에서 제거한다.
취소 버튼: 아니오. 다이얼로그를 닫고 채팅방 목록으로 돌아간다.

그룹채팅 초대 리스트 재사용

각 그룹채팅에서도 파티 생성 완료 화면과 동일한 동네 초대 가능 사용자 리스트를 사용한다. 채팅방 안에서는 상단 초대 버튼 또는 멤버 목록의 초대 탭에서 진입한다.

동일 정책: 동네 기준, 초대 허용 여부, 접근 조건 충족 여부, 초대 거절 3회 차단 정책은 생성 완료 화면과 동일하게 적용한다.
권한: 공개파티는 모든 멤버가 초대할 수 있다. 비공개파티는 파티장이 초대 권한을 멤버에게 허용한 경우에만 멤버가 초대할 수 있다.
중복 방지: 이미 참여 중이거나 이미 초대된 사용자는 후보에서 제외하거나 초대 완료 상태로 표시한다.

그룹채팅 온라인 상태 표시

그룹채팅에서는 멤버가 현재 온라인인지 초록불로 표시한다. 이는 실시간 위치 공유가 아니라 채팅 접속/활동 상태 표시이며, 사용자의 정확한 위치나 상세 활동 시간은 노출하지 않는다.

표시 위치 표시 방식 정책
채팅 메시지 아바타 프로필 이미지 하단에 작은 초록 점 현재 온라인인 멤버에게만 표시한다.
멤버 목록 닉네임 옆 초록 점과 "온라인" 라벨 온라인 멤버를 상단 정렬할 수 있다.
파티장 Live 요약 현재 입장 인원과 별도로 온라인 인원 표시 예: 5명 입장 · 3명 온라인
온라인 기준: 앱이 포그라운드에 있거나, 해당 파티 채팅/Live Activity를 최근 짧은 시간 안에 활성화한 사용자를 온라인으로 본다.
오프라인 전환: 일정 시간 동안 활동 신호가 없으면 자동으로 오프라인 처리한다.
프라이버시: 마지막 접속 시각, 현재 위치, 다른 파티에서의 활동 여부는 표시하지 않는다.

그룹채팅 시스템 메시지 표시

참여, 퇴장, 상태 변경 같은 시스템 메시지는 일반 사용자 메시지 말풍선으로 표시하지 않는다. 카카오톡 그룹채팅처럼 채팅 흐름 중간에 플레인한 시스템 텍스트로 표시한다.

이벤트 표시 문구 표시 방식
파티 합류 {닉네임}님이 파티에 합류했어요! 말풍선 없이 중앙 정렬 또는 흐름 내 플레인 텍스트
파티 나가기 {닉네임}님이 파티를 나갔습니다. 말풍선 없이 중앙 정렬 또는 흐름 내 플레인 텍스트
파티 상태 변경 파티 상태가 {모집 중/집결 중/이동 중/종료}으로 변경됐어요. 말풍선 없이 플레인 텍스트, 필요 시 작은 상태 배지 포함
초대 URL 생성 파티장이 즉시 입장 초대 URL을 만들었어요. 파티장에게만 보이거나 채팅방 전체 노출 여부를 설정으로 제어
시각 스타일: 작은 글자, 낮은 대비, 배경 없는 텍스트를 기본으로 한다. 사용자 아바타, 닉네임 헤더, 말풍선 꼬리는 사용하지 않는다.
타임라인 역할: 시스템 메시지는 파티의 주요 상태 변화를 기록하지만 대화보다 시각적 우선순위가 낮아야 한다.

그룹채팅 운영 기능

기능 동작 정책
강퇴 방장이 멤버 프로필 또는 멤버 목록에서 강퇴를 선택한다. 강퇴된 사용자는 파티에서 나가고 그룹채팅 접근 권한을 잃는다. 채팅에는 "{닉네임}님이 파티에서 내보내졌습니다." 시스템 메시지를 남긴다.
방장 넘기기 방장이 특정 파티원 프로필 또는 멤버 목록에서 방장 넘기기를 선택하고 확인한다. 선택한 파티원의 role이 host가 되고 기존 방장은 member가 된다. Party.host_user_id를 새 방장으로 갱신하고, 채팅에는 "{닉네임}님이 방장이 되었습니다." 시스템 메시지를 남긴다.
메시지 가리기 방장이 특정 메시지를 길게 눌러 가리기를 선택한다. 메시지 본문 대신 "방장이 가린 메시지입니다."를 표시한다. 원문은 신고/운영 확인을 위해 서버에 보존한다.
레이블 노출: 멤버 프로필 화면에는 "방장관리" 같은 섹션 레이블을 노출하지 않는다. 방장에게만 강퇴, 방장 넘기기 등의 액션 버튼 또는 액션 메뉴를 보여준다.
위임 확인: 방장 넘기기는 되돌리기 어려운 권한 변경이므로 확인 다이얼로그를 거친다.

참가자 프로필 확인

참가자는 그룹채팅 안에서 서로의 프로필을 눌러 파티용 프로필을 확인할 수 있다. 계정 신원과 파티 프로필은 분리하며, 실명/이메일/카카오 계정 식별자는 기본 노출하지 않는다.

진입: 채팅 메시지의 아바타, 멤버 목록, 온라인 멤버 표시에서 프로필을 열 수 있다.
노출 정보: 닉네임, 프로필 이미지, 역할, 온라인 여부를 보여준다. 인증 조건 충족 여부는 입장 검증에만 사용하고 프로필 칩으로 노출하지 않는다.
칩 제거: "제한 없음", "파티용 프로필"처럼 사용자가 해석하기 어려운 칩은 프로필 화면에서 제거한다.
방장 추가 액션: 방장은 프로필 화면에서 강퇴, 방장 넘기기 같은 액션을 볼 수 있다. 단, 별도 "방장관리" 레이블은 표시하지 않는다.

알림 설정 정책

알림 설정은 전역 카테고리 설정과 채팅방별 설정을 분리한다. 전역 카테고리가 OFF이면 해당 하위 설정은 UI에서 dimmed 처리해 현재 알림이 오지 않는 이유를 명확히 보여준다.

알림 분류 사용자 설정 포함 내용
그룹채팅 알림 ON/OFF 새 메시지, 멘션, 파티 내 대화 알림. OFF이면 각 채팅방의 알림 토글은 dimmed 처리한다.
파티초대 알림 ON/OFF 즉시 입장 초대 URL 수신, 초대 만료, 초대 거절 관련 알림.
입장 요청/승인 알림 ON/OFF 파티장에게 온 입장 요청, 내가 보낸 입장 요청의 승인/거절 결과.
시스템 알림 항상 수신 계정, 보안, 신고/제재, 인증 만료, 파티 운영에 필요한 필수 알림. OFF를 제공하지 않는다.
홍보/이벤트 알림 ON/OFF 이벤트, 혜택, 추천 파티 등 마케팅성 알림.

전역 OFF와 하위 설정 표시

그룹채팅 알림 OFF: 채팅방별 알림 토글은 dimmed 처리하고 "전역 그룹채팅 알림이 꺼져 있어요" 문구를 표시한다.
파티초대 알림 OFF: 초대 관련 세부 알림 설정은 dimmed 처리한다. 단, 초대 URL 자체의 유효성 확인은 앱 진입 시 계속 수행한다.
입장 요청/승인 알림 OFF: 푸시 알림은 보내지 않지만 앱 안의 요청함/배지는 유지한다.
시스템 알림: 필수 알림이므로 dimmed된 항상 수신 상태로 보여주고 토글을 제공하지 않는다.
Live Activity 표시: 채팅 알림과 별도 설정이다. 채팅 알림을 꺼도 사용자가 허용하면 Live Activity형 파티 상태는 유지할 수 있다.

예외 케이스

같은 위치 다중 파티: 같은 POI 또는 가까운 좌표에 여러 파티가 있으면 하나의 위치별 파티 목록 패널 안에 최신 live 파티부터 정렬한다.
좌표와 POI 차이: 네이버 POI가 있으면 POI 기준으로 묶고, POI가 없는 지도 클릭은 좌표 반경 기준으로 묶는다.
파티 생성 한도: 한 사용자는 지도에 노출되는 파티를 최대 3개까지만 생성할 수 있다. 집계 기준은 host_user_id가 현재 사용자이고 map_visibility_status가 visible인 파티다.
한도 제외: map_visible_until이 지나 지도에서 더 이상 발견되지 않는 파티, map_visibility_status가 hidden인 파티, 사용자가 참여만 한 파티는 생성 한도 집계에서 제외한다.
한도 초과 안내: 이미 3개가 노출 중이면 "지도에 노출 중인 파티는 최대 3개까지 만들 수 있어요"를 보여주고, 내 파티 목록으로 이동해 종료/정리할 수 있게 한다.
인증 미충족: 파티 요약은 보여주되 멤버 상세와 채팅은 입장 전까지 노출하지 않는다.
초대 거절 3회: 같은 사용자가 동일한 파티 초대를 3번 거절하면 해당 파티의 초대를 다시 받지 않는다. 이후 파티장이 초대 URL을 보내려 해도 대상자 선택 또는 발송 단계에서 제외한다.
호스트 이탈: MVP에서는 호스트 탈퇴를 막고 파티 종료 후 나갈 수 있게 한다.
안전: 실명, 이메일, 카카오 계정 식별자는 파티 화면과 채팅에 직접 노출하지 않는다.

7. MVP 개발 우선순위

우선순위 개발 단위 이유
P0 카카오 로그인, 사용자 세션, 기본 프로필 모든 파티/채팅/인증 흐름의 기반이다.
P0 좌표/POI 기반 파티 생성, 즉시 입장 초대 URL, 파티 참여 지도에서 위치를 선택하고 그 위치에 파티를 여는 기본 루프다.
P0 지도 클릭, 위치별 파티 목록, 파티 입장 특정 좌표에서 열린 라이브 파티를 발견하는 핵심 차별점이다.
P0 그룹채팅과 위치/참여 시스템 메시지 파티가 살아 있다는 느낌을 만드는 핵심 경험이다.
P1 선택 인증과 인증 조건 입장 파티 성격에 맞는 신뢰와 소속감을 만든다.
P1 그룹채팅 운영: 강퇴, 방장 넘기기, 메시지 가리기, 참가자 프로필 확인 그룹채팅 운영과 안전한 파티 진행을 지원한다.
P1 파티 상태 변경: 모집 중, 집결 중, 이동 중, 종료 위치 기반 파티의 현재 상황을 빠르게 파악하게 한다.
P2 신고, 차단, 인증 만료 운영 안정성을 위해 필요하지만 초기 검증 이후 확장한다.

8. 1차 MVP 의사결정

첫 MVP 타깃: 특정 집단으로 과하게 좁히지 않는다. 공연, 여행, 맛집 탐방, 축제, 번개 약속처럼 위치와 현재성이 강한 소규모 모임 전반을 대상으로 검증한다.
대표 사용 시나리오: 사용자가 특정 위치에 열린 파티를 발견하거나 직접 만들고, 공개/비공개 입장 조건에 따라 채팅으로 합류하는 상황을 1차 유스케이스로 둔다.
시간성 포지션: 초반 사용자는 이 서비스를 미래 약속을 쌓아두는 캘린더가 아니라 지금 살아 있는 파티방으로 인식해야 한다. MVP에서는 오늘 또는 곧 움직이는 파티를 기본값으로 설계한다.
핵심 UI 킥: 서비스의 차별점은 Live Activities처럼 앱 밖에서도 파티가 살아 있는 상태로 느껴지는 UI다. 파티장은 현재 입장 인원과 입장 요청 수를 보고, 초대받은 사용자는 초대받은 파티명을 먼저 확인한다.
MVP 입력 방식: 사용자는 활동 유형을 선택하지 않는다. 지도에서 좌표 또는 POI를 선택하고, 파티 제목과 공개/비공개 여부를 정한다. 활동 유형은 내부 분석 값으로만 축적한다.
입장 정책: 파티는 공개파티와 비공개파티로 나눈다. 공개파티는 즉시 입장한다. 비공개파티는 파티장 승인, 비밀번호 입력, 즉시 입장 초대 URL 중 파티장이 허용한 방식으로 입장한다.
접근 조건: 공개/비공개와 별개로 파티장은 제한 없음, 내가 인증한 대학교만 허용, 직장 인증 사용자만 허용 중 하나를 선택할 수 있다.
초대 피로도 방지: 동일 사용자가 같은 파티의 초대를 3번 거절하면 해당 파티의 초대를 더 이상 받지 않는다. 이 차단은 해당 파티에만 적용된다.
호스트 권한: 1차에서는 파티장이 입장 요청 승인/거절, 비밀번호 설정, 즉시 입장 초대 URL 생성, 파티 프로필 사진 변경, 파티 상태 변경, 파티 종료를 담당한다.
채팅 구현: MVP부터 실시간 WebSocket 기반으로 간다. 이 서비스에서 채팅은 부가 기능이 아니라 라이브 파티가 살아 있다는 감각을 만드는 핵심 경험이다.
지도 노출 정책: 지도에서는 오늘 활동 중인 파티를 우선 노출한다. 오늘이 지난 파티는 map_visible_until 이후 지도에서 발견되지 않지만, 참여자의 그룹채팅 목록과 채팅 기록은 유지한다.
생성 한도: 한 사용자가 지도에 노출 중인 파티는 최대 3개까지 만들 수 있다. 지도에서 내려간 파티와 참여만 한 파티는 집계하지 않는다.
네이버 POI 저장 정책: 우리 DB에는 naver_poi_id, 장소명, 주소, 좌표, 카테고리, 파티별 상태만 저장한다. 리뷰, 별점, 이미지 등 네이버가 소유한 풍부한 콘텐츠는 저장하지 않고 필요 시 원천 화면으로 연결한다.
조직 인증: 인증은 서비스 전체 타깃을 좁히는 장치가 아니라 파티별 접근 조건 파라미터다. 학교/회사 이메일 인증은 파티장이 필요할 때만 요구한다.
공개 범위: 1차에서 지도에는 공개파티와 비공개파티가 모두 노출될 수 있다. 단, 비공개파티는 입장 전 채팅과 멤버 상세를 제한한다.
첫 화면 전략: 가입 직후에는 현재 지도 주변의 라이브 파티와 "이 위치에 파티 만들기"를 전면에 둔다. 내 파티가 생긴 뒤부터 홈의 중심을 진행 중 파티 목록으로 바꾼다.
보류 결정: 미래 일정 관리, 실시간 멤버 위치, 장소 자동 추천, 반복 파티, 복잡한 조직별 추천은 MVP 검증 이후로 미룬다.
