Service Strategy Draft

지도 기반 파티 서비스 상위기획

네이버 지도 위 특정 좌표나 POI에서 지금 열린 파티를 발견하고, 공개/비공개 조건에 따라 입장해 함께 움직이는 로컬 라이브 소셜 서비스.

핵심 관점

이 서비스는 장소를 찾는 도구가 아니라, 사용자가 자기 주변의 사람들과 함께 움직이는 사람이라고 느끼게 만드는 지도 기반 소셜 레이어다.

상위 철학 서비스 구조 도메인 모델 MVP 범위

1. 상위 철학

사람들이 서비스의 팬이 되는 구조는 단순히 "서비스가 유용한가"에서 끝나지 않는다. 더 중요한 질문은 이 서비스를 쓰는 내가 어떻게 보이는지, 어떤 집단에 속한다고 느끼는지, 그리고 이 서비스를 켰을 때 어떤 사람이 된 느낌을 받는지다.

혼자 보던 지도를, 같이 움직이는 지도로 바꾼다.

사용자가 얻어야 하는 자기 이미지

감각 있는 사람

좋은 위치에서 지금 열릴 만한 움직임을 만들고, 사람들의 합류를 돕는 사람.

좋은 호스트

그냥 방을 만드는 사람이 아니라 오늘의 움직임을 여는 사람.

연결된 사람

내 학교, 회사, 동네, 취향 안에서 가볍게 함께 움직이는 사람.

팬이 되는 구조를 위한 질문

정체성: 이 기능은 사용자를 더 센스 있는 사람처럼 느끼게 하는가?
소속감: 첫 30초 안에 "내 사람들, 내 동네, 내 취향"이 보이는가?
마찰: 지도에서 파티를 발견하고 들어가기 전의 민망함과 부담을 줄이는가?
사회성: 관계보다 활동을 먼저 내세워 대화와 참여의 부담을 낮추는가?
신뢰: 인증 기반 안전감을 주되 감시나 신원 노출처럼 느껴지지 않게 하는가?

2. 서비스 한 줄 정의

인증된 소속과 취향을 바탕으로, 사람들이 지도 위 특정 위치에서 지금 열린 파티를 발견하고 부담 없이 합류하게 하는 지도 기반 로컬 라이브 소셜 플랫폼.

핵심 컨셉

서비스의 중심 단위는 파티다. 파티는 특정 좌표나 POI에 붙어 있으며, 사용자를 공연, 여행, 맛집 탐방, 축제, 번개 약속 같은 하나의 공통 목적 아래 묶는다. 각 파티는 하나의 그룹채팅을 가지며, 지도에서 발견한 현재성은 Live Activities형 UI와 채팅 타임라인으로 이어진다.

지도 탐색 목적 기반 파티 그룹채팅 인증 기반 신뢰 파티별 프로필 위치별 라이브 파티 Live Activities형 UI

3. 지도 전략

네이버 지도는 지도 렌더링, POI 클릭 이벤트, 좌표와 기본 장소 정보 확보용 인프라로 사용한다. 서비스 경험의 의미 계층은 우리 DB가 소유한다.

네이버 지도 역할

  • 기본 지도 렌더링
  • POI 클릭 이벤트
  • 좌표, 장소명, 주소 등 기본 정보 확보
  • 지도 탐색 인터랙션 제공

우리 서비스 역할

  • 좌표/POI별 라이브 파티 목록
  • 파티 기준 위치와 현재 상태
  • 채팅 시스템 메시지
  • 멤버 프로필과 인증 배지
  • 공개/비공개 파티 접근 정책
  • Live Activities형 실시간 요약

POI 클릭 후 기본 흐름

  1. 사용자가 네이버 지도 위 POI를 클릭한다.
  2. 클라이언트가 POI 식별자, 좌표, 이름, 주소를 수집한다.
  3. 우리 서버가 Place 데이터를 조회하거나 생성한다.
  4. 해당 POI 또는 좌표 반경에 생성된 active/live 파티를 조회한다.
  5. 위치별 파티 목록 드래그 확장 패널을 우리 서비스 데이터 기준으로 보여준다.
  6. 사용자는 공개파티에 바로 입장하거나, 비공개파티에 승인 요청/비밀번호/초대 URL로 입장한다.

4. 핵심 도메인

파티

파티는 사람, 목적, 기준 위치, 채팅, 입장 정책, 라이브 상태를 묶는 최상위 단위다. 사용자는 특정 좌표나 POI에 파티를 생성해 파티장이 되거나, 지도에서 발견한 공개파티에 바로 입장하거나, 비공개파티에 승인/비밀번호/초대 URL로 조인할 수 있다.

Party - id - title - internal_activity_type - host_user_id - anchor_place_id - anchor_lat - anchor_lng - anchor_label - visibility: public | private - join_policy: instant | approval_required | password - access_condition_type: none | same_school | company_verified - required_organization_id - status - live_status - current_member_count - pending_join_request_count - created_at PartyMember - party_id - user_id - role: host | member - display_name - profile_image_url - joined_at - left_at

그룹채팅과 Live Activities형 요약

하나의 파티는 하나의 그룹채팅을 가진다. 채팅은 단순 대화방이 아니라 파티의 참여, 입장 요청, 상태 변화가 쌓이는 타임라인이다. 파티장은 현재 입장 인원과 입장 요청 수를 확인하고, 초대받은 사용자는 초대받은 파티명을 먼저 확인한다.

Party 1 : 1 ChatRoom Party 1 : N PartyMember ChatRoom 1 : N ChatMessage ChatMessage.type - text - system - location

장소와 파티 기준 위치

전역 장소인 Place와 파티가 열리는 기준 위치를 분리한다. MVP에서는 여러 장소 후보를 올려 투표하는 구조가 아니라, 지도에서 클릭한 좌표나 POI에 파티가 붙고 그 위치에 열린 파티 목록을 보여주는 구조를 우선 검증한다.

Place - id - naver_poi_id - name - address - lat - lng - category Party.anchor - anchor_place_id - anchor_lat - anchor_lng - anchor_label

5. 프로필과 신원 인증

서비스는 신뢰를 위해 신원 인증을 받지만, 파티 안에서는 오픈채팅처럼 가벼운 표시 프로필로 활동할 수 있게 한다. 계정 신원과 파티 프로필은 분리한다.

인증 방식

  • 카카오 인증: KakaoTalk SNS Login 활용
  • 학교 인증: 학교 도메인 이메일 인증
  • 직장 인증: 회사 도메인 이메일 인증, MVP 이후 확장
  • 한 사용자는 여러 인증을 가질 수 있음

노출 정책

  • 파티 안에서는 닉네임과 프사 중심 노출
  • 실제 이메일, 실명, 카카오 계정 정보는 기본 비노출
  • 인증 배지와 같은 조직 여부는 제한적으로 노출 가능
  • 신고와 제재를 위해 내부 신원은 유지
VerifiedIdentity - user_id - type: kakao | school_email | company_email - provider - verified_email - organization_id - verified_at - status PartyMemberProfile - party_id - user_id - display_name - profile_image_url - profile_source: default | custom

6. 주요 사용자 플로우

파티 생성

  1. 사용자가 지도에서 파티를 열 좌표 또는 POI를 누른다.
  2. 해당 위치의 파티 목록 드래그 확장 패널에서 이 위치에 파티 만들기를 선택한다.
  3. 파티 제목과 공개/비공개 여부, 입장 방식, 접근 조건을 설정한다.
  4. 접근 조건은 제한 없음, 내가 인증한 대학교만 허용, 직장 인증 사용자만 허용 중 선택한다.
  5. 파티와 그룹채팅이 함께 생성된다.
  6. 필요하면 즉시 입장 초대 URL을 만들고 멤버를 초대한다.

파티 참여

  1. 사용자가 지도에서 특정 좌표나 POI를 클릭한다.
  2. 해당 위치에 생성된 라이브 파티 목록을 확인한다.
  3. 파티의 접근 조건을 확인한다.
  4. 파티용 닉네임과 프사를 설정한다.
  5. 공개파티는 바로 입장하고, 비공개파티는 승인/비밀번호/초대 URL로 입장한다.
  6. 파티에 참여하고 그룹채팅에 자동 입장한다.

위치별 파티 발견

  1. 지도에서 POI를 클릭한다.
  2. 해당 POI 또는 좌표 반경의 active/live 파티를 조회한다.
  3. 파티 제목, 공개/비공개 여부, 현재 입장 인원, 생성 시각을 보여준다.
  4. 초대받은 사용자는 초대받은 파티명을 먼저 확인한다.
  5. 파티장은 현재 입장 인원과 입장 요청 수를 Live Activities형 UI로 확인한다.
  6. 파티가 종료되거나 만료되면 지도 목록에서 내려간다.

7. 첫 30초 설계

첫 30초 안에 사용자는 기능 설명이 아니라 자기 관련성을 느껴야 한다. "여기 내 사람들이 있네", "여기 내 동네가 있네", "여기 내가 할 수 있는 일이 있네"가 보여야 한다.

바로 잡아야 할 맥락

  • 어디 소속인가: 학교, 회사, 카카오 인증
  • 어디에서 움직이는가: 현재 위치, 동네, 자주 가는 지역
  • 무엇을 하고 싶은가: 공연, 여행, 맛집 탐방, 축제, 번개 약속

첫 화면에서 보여줄 것

  • 내 주변에서 열린 파티
  • 내 인증 소속 기반 파티
  • 지도에서 선택한 위치의 라이브 파티
  • 지금 바로 만들 수 있는 위치 기반 파티
피해야 할 것: 빈 지도만 보여주기, 기능 설명 카드 나열, 맥락 없는 검색 유도, "파티를 생성하세요" 같은 행정적인 첫 액션.

8. MVP 범위

1차 MVP

  • 카카오 로그인
  • 선택 인증: 학교/회사 이메일 인증 구조
  • 기본 프로필 설정
  • 좌표/POI 기반 파티 생성과 참여
  • 공개파티 즉시 입장
  • 비공개파티 승인/비밀번호/즉시 입장 초대 URL
  • 파티별 닉네임/프사 설정
  • 파티별 그룹채팅
  • 네이버 지도 표시와 POI 클릭
  • 위치별 라이브 파티 목록
  • Live Activities형 파티 요약 UI

후순위 기능

  • 장소 후보 추가와 투표
  • 조직별 파티 추천
  • 실시간 멤버 위치
  • 러닝 경로 설계
  • 일정 조율
  • 장소 자동 추천
  • 반복 파티
  • 신고, 차단, 강퇴
  • 인증 만료와 재인증

9. 현재까지의 설계 원칙

파티가 서비스의 중심 단위다.
하나의 파티는 하나의 그룹채팅을 가진다.
지도는 네이버 지도 인프라를 쓰되, 서비스 상태는 우리 DB가 소유한다.
Place와 파티 기준 위치를 분리한다.
계정 신원과 파티 프로필을 분리한다.
사용자는 인증된 사람이지만, 파티 안에서는 가벼운 프로필로 활동한다.
파티 접근 권한은 공개/비공개 입장 방식과 학교/직장 인증 기반 접근 조건을 조합해 제어한다.
채팅은 대화뿐 아니라 파티 참여와 상태 변화의 기록 타임라인이다.
Live Activities형 UI는 파티가 살아 있다는 감각을 만드는 핵심 차별점이다.
기능 판단 기준은 효용뿐 아니라 사용자 정체성, 소속감, 행동 마찰이다.
